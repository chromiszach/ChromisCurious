B. How to Upload a Theme to Your Ghost Blog - top

1. Upload a Theme to Self Hosted Ghost

  • 1. Locate the theme directory within the download files. It can be found at theme/curious.
  • 2. Login to FTP/SFTP of your blog, or the file browser of your provided hosting service.
  • 3. Navigate to the /content/theme directory within your Ghost installation.
  • 4. Upload the curious directory found in step one into the remote /content/theme directory.
  • 5. Restart your Ghost service. The process for this will change dependent on your provider. If you are unaware how to restart the service please raise a support issue with your host.
  • 6. Login to your Ghost admin panel. This can be found at yourdomain.com/ghost.
  • 7. Navigate to the Settings > General tab.
  • 8. In the Theme drop down, select the newly installed Curious theme. Click Save to save the theme changes.

2. Upload a Theme to Ghost.io

  • 1. Login to your Ghost.org user panel. Visit Ghost.org and enter your login credentials.
  • 2. From your Ghost(Pro) dashboard, select the Settings (“gear”) icon to the right of the “Write a Post” button on the blog you wish to upload the theme.
  • 3. In your blog’s control panel, you can either “drag & drop” curious theme zip file into the “Add a Theme” drop zone, or click on the drag & drop zone to launch the traditional file upload prompt.
  • 4. Upload the theme and Save All Blog Settings. You may have to activate the theme in your ghost admin panel.

C. Theme Features - top

1. Favicon

To change the favicon, replace favicon.ico file located in the assets folder (Ghost/content/themes/curious/assets) of the theme.

2. Navigation

You can add, edit, delete and reorder menu links on your Ghost blog, directly from the navigation tool within the blog admin area, located at /ghost/settings/navigation/.

To include a static page on your navigation menu, first type the name of the page as you’d like it to appear on your menu in the label field.

Next, click inside the URL field of the menu item. The blog URL will auto-populate with http://yourdomain.com/. You will need to add in the page slug after the final /. Once satisfied with your page link, click the blue Save button to add the page to the navigation menu.

3. Static Pages

In order to create a static page you create a new post, just like you would any other post. Once you have opened up the new post, there is a cog wheel icon next to where it says "Save Draft" or "Update Post" depending on if you have published the post or not. Click on that cog, and check the "Turn this post into a static page" box. This will convert your post to a static page.

4. Cover Images

You can add cover images to all of your posts, static pages, tag and author pages. Open your post/page settings menu, drag and drop or locate the image on your computer, save it and you're all set.

5. Intro Section

The intro section is displayed on the homepage, right below the site navigation. You can edit the information displayed in the intro section in the home.hbs file located in the main theme folder (Ghost/content/themes/curious). By default, the blog title, blog description, blog cover image and the "Subscribe" button are displayed. If the "Subscribers" feature is enabled in Ghost, the "Subscribe" button leads to the Ghost subscribe page. If not, the button leads to the RSS feed page.

6. Subscribers

In version 0.8.0 Ghost introduced the "Subscribers" feature. To enable this feature, visit yourblog.com/ghost/settings/labs/ and tick the "Subscribers" checkbox. When the "Subscribers" feature in labs is enabled, the subscription form is displayed below the post content in the single post page, the optional Subscribe widget is displayed in the theme sidebar and the "Subscribe" button in the intro section of the homepage leads to the Ghost subscribe page.

7. Syntax Highlighter

Curious comes with Rainbow.js plugin to automatically syntax highlight code blocks in your posts. To make syntax highlighter work, just wrap the code in <pre><code data-language="css"></code></pre> tags and change the value of "data-language" attribute to your code language. Supported languages include CSS, Html, JavaScript, Python, Ruby.

8. Image and Blockquote Styles

When adding blockquotes and images to your posts and pages, you can use special CSS classes to create a more attractive flow of content.

For instance, you can align your image left or right, by wrapping it in a p tag with the CSS class alignleft or alignright like this:

<p class="alignleft">
![alternative text](/content/images/2015/09/my_image.jpg)
</p>
<p class="alignright">
![alternative text](/content/images/2015/09/my_image.jpg)
</p>

You can also make your image wider than the content by wrapping it in a p tag with the CSS class full-width like this:

<p class="full-width">
![alternative text](/content/images/2015/09/my_image.jpg)
</p>

alignleft or alignright CSS classes can be applied to blockquotes as well. See an example below.

<blockquote class="alignleft">
<p>Good ideas never go out of style.</p>
</blockquote>
<blockquote class="alignright">
<p>Good ideas never go out of style.</p>
</blockquote>

9. Disqus Comments

Curious theme comes with Disqus commenting built-in, you just need to have a Disqus account (create it at Disqus.com). To enable commenting, you need to change the default Disqus account shortname "my_disqus_shortname" on line 19 in the default.hbs file located in the main theme folder (Ghost/content/themes/curious).

disqus_shortname = 'my_disqus_shortname', // add your Disqus shortname

If you wish to disable commenting, just remove all comments related code in the comments.hbs file located in the partials folder (Ghost/content/themes/curious/partials) of the theme.

10. Sidebar

The theme sidebar contains Ghost subscription form, hardcoded text widgets and hardcoded MailChimp newsletter subscription form. The sidebar.hbs file is located in the partials folder (Ghost/content/themes/curious/partials) of the theme.

Text Widgets

You will find to hardcoded text widgets "About" and "Contacts" in the sidebar of the theme. To change the "About" widget information, edit the text on line 8 in the sidebar.hbs file located in the partials folder (Ghost/content/themes/curious/partials). To change the "Contact" widget information, edit the text on line 28 in the sidebar.hbs file located in the partials folder (Ghost/content/themes/curious/partials). If you do not need those widgets, simply delete the aside elements:

<aside class="widget widget-text">
  <h2 class="widget-title">About {{@blog.title}}</h2>
  <p>Lorem ipsum dolor sit amet...</p>
</aside><!-- .widget -->
<aside class="widget widget-text">
  <h2 class="widget-title">Contact</h2>
  <p>Lorem ipsum dolor sit amet...</p>
  <ul class="contacts">
    <li><span class="icon icon-envelope" aria-hidden="true"></span> blog[at]gmail.com</li>
    <li><span class="icon icon-map-marker" aria-hidden="true"></span> 123 Some Street</li>
    <li><span class="icon icon-phone" aria-hidden="true"></span> +123 112 23445</li>
  </ul>
</aside><!-- .widget -->

Ghost Subscription Form

If you have enabled the "Subscribers" feature in labs, Ghost subscription form widget is displayed in the sidebar.

MailChimp Newsletter Subscription Form

If you want to integrate MailChimp newsletter subscription form, there is a hardcoded Mailchimp newsletter widget in the sidebar. To enable MailChimp newsletter subscriptions, you need your unique form action URL.

To get the form action URL, first log in to your MailChimp account. From your account Dashboard, click Lists in the navigation menu. Find the list you want to connect to your website or create a new list. Click the drop-down menu next to your list and choose Signup forms. Select Embedded forms. On the Embedded forms page click Naked. Find the Copy/paste onto your site section and click anywhere in the box to select the code. Copy the code and paste it in a text editor. Find the form action URL, which is a unique URL that you need to link your website to MailChimp. Select the URL between the quotation marks after the code action=.

Copy that URL and paste it on line 16 in the sidebar.hbs file located in the partials folder (Ghost/content/themes/curious/partials).

If you use Ghost subsription or some other newsletter service and do not need MailChimp subscription form, simply delete the aside element:

<aside class="widget widget-newsletter">
  <h2 class="widget-title">{{@blog.title}} Newsletter</h2>
  <p>Enter your email address to subscribe to this blog and receive notifications of new posts by email.</p>
  <!-- Begin MailChimp Signup Form -->
  <div id="mc_embed_signup">
    <form action="{{! Add your form action URL }}" method="post" id="mc-embedded-subscribe-form" name="mc-embedded-subscribe-form" class="validate" target="_blank" novalidate>
      <div id="mc_embed_signup_scroll">
        <label for="mce-EMAIL" class="screen-reader-text">Email Address</label>
        <input type="email" value="" name="EMAIL" class="email" id="mce-EMAIL" placeholder="email address" required>
        <div class="clear"><input type="submit" value="Subscribe" name="subscribe" id="mc-embedded-subscribe" class="button"></div>
      </div>
    </form>
  </div>
  <!--End mc_embed_signup-->
</aside><!-- .widget -->

11. Footer Widgets

The theme footer widget area contains "Latest Posts", "Instagram" and "Tags" widgets. The footer.hbs file is located in the partials folder (Ghost/content/themes/curious/partials) of the theme.

Latest Posts Widget

There is currently no way to create a latest posts widget in Ghost. So for now we've implemented a temporary solution - the latest posts list is generated automatically from the RSS feed.

The widget displays 5 posts by default, but you can change the number of posts on line 22 in the default.hbs file located in the main theme folder (Ghost/content/themes/curious).

latest_posts_count = 5; // change the latest posts number

Instagram Widget

To display Instagram photos you need to change the default instagram_user_id and instagram_access_token on lines 20, 21 in the default.hbs file located in the main theme folder (Ghost/content/themes/curious).

instagram_user_id = '2372162651',
instagram_access_token = '2372162651.d4daff6.d6c91752b4914a8da6e01a85747ae89e',

To get your user ID go to http://jelled.com/instagram/lookup-user-id or http://www.otzberg.net/iguserid/ and enter your Instagram username.

To obtain an Access Token first you need to get a Client ID. Go to Instagram Developer page and log in. Click on Register your application. If this is your first application, register as developer by filling the information asked, accept the API terms of use and brand guidelines and click Sign up button. Click Register your Application again to proceed and then click Register a New Client at the top right. Set application name to your website name, type in a description, add the url of your site in the "Website URL" and "Valid redirect URIs" fields and enter your email. Click "Security" tab and uncheck the box "Disable implicit OAuth". Hit register.

Now, paste the following url into your web browser. Be sure to replace CLIENT-ID and REDIRECT-URI with your Client ID and Redirect URI you received when you registered your application in the previous step.

https://api.instagram.com/oauth/authorize/?client_id=CLIENT-ID&redirect_uri=REDIRECT-URI&response_type=token

Hit enter, and you will connect to Instagram's servers. Instagram will ask you whether you want to grant the client you registered access to your account. Once your client has been granted access to your account, Instagram will redirect you to your Redirect URI with the access_token in the url fragment. It will look like this:

http://your-redirect-uri#access_token=ACCESS-TOKEN

Simply grab the access_token off the URL fragment and add it to the default.hbs file.

Tag Cloud Widget

There is currently no way to create a tag cloud in Ghost. So for now we've implemented a temporary solution - the tag cloud is generated automatically from the RSS feed.

Important! You should leave the default, automatically generated, URL for the tag (in the Tag Settings). If you change it, the tag link will become broken.
